# 功能規格書：前端工程師專業個人基地

**功能分支**: `001-frontend-portfolio`  
**建立日期**: 2026-03-06  
**狀態**: 草稿 (Draft)  
**輸入需求**: "建立一個具備專業品牌感的前端工程師個人基地，整合以下三大核心內容： 1. 工作經驗: 簡略呈現個人履歷，以及個人資料 (線上履歷連結、LinkIn 連結、github 連結等等)。 2. 作品集: 展示個人開發的各個 side project。 3. 技術部落格: 分享前端開發和 AI 應用等經驗與深度技術探討。"

## 使用者情境與測試 _(必填)_

### 使用者故事 1 - 專業個人檔案與連結 (優先級: P1)

使用者進入網站後，能快速了解工程師的專業背景、瀏覽工作經驗摘要，並能輕易取得外部專業平台連結（履歷、LinkedIn、GitHub）。

**為何設定此優先級**: 建立專業形象並提供聯絡/專業連結是個人網站最首要的目標。

**獨立測試方式**: 進入首頁後，驗證個人簡介區塊是否正確顯示專業摘要，且所有對外連結（履歷、LinkedIn、GitHub）是否能正確連到對應的目標頁面。

**驗收情境**:

1. **假設 (Given)** 訪客進入首頁，**當 (When)** 他們查看個人簡介區塊時，**則 (Then)** 他們能看到專業摘要以及履歷、LinkedIn 與 GitHub 的連結。
2. **假設 (Given)** 訪客點擊 LinkedIn 或 GitHub 連結，**當 (When)** 觸發點擊動作時，**則 (Then)** 各自的外部頁面會在新分頁中正確開啟。

---

### 使用者故事 2 - 作品集展示 (優先級: P1)

使用者透過瀏覽作品集區塊，能視覺化地探索工程師開發的各項 side project，了解其使用的技術堆疊 (Tech Stack)，並明確被引導前往專案的 GitHub Repo 或 Live Demo。

**為何設定此優先級**: 透過 side project 展示實作能力是建立前端工程師個人品牌的重要一環。

**獨立測試方式**: 導航至作品集區塊，驗證至少有一個專案被正確顯示，包含其詳細資訊與可正常運作的連結。

**驗收情境**:

1. **假設 (Given)** 訪客導航至作品集/專案區塊，**當 (When)** 他們查看專案列表時，**則 (Then)** 他們能看到每個專案的縮圖、標題、描述與使用的技術堆疊。
2. **假設 (Given)** 訪客查看特定專案，**當 (When)** 他們點擊專案的 GitHub Repo 或 Live Demo 連結時，**則 (Then)** 會正確跳轉並引導至該專案的外部網址。

---

### 使用者故事 3 - 技術部落格 (優先級: P2)

使用者可以閱讀針對前端開發、AI 應用與其他技術經驗分享的深度技術文章。

**為何設定此優先級**: 部落格能建立技術影響力與知識深度，但在優先級上為核心個人介紹與作品集展示之後的次要但高價值的附屬功能。

**獨立測試方式**: 驗證部落格列表是否顯示文章摘要，且點擊單篇文章後能正確開啟並顯示全文內容。

**驗收情境**:

1. **假設 (Given)** 訪客導航至部落格區塊，**當 (When)** 頁面載入時，**則 (Then)** 會顯示文章摘要列表（包含標題、日期、如 "Frontend" 或 "AI" 的標籤）。
2. **假設 (Given)** 訪客點擊特定部落格文章，**當 (When)** 頁面載入時，**則 (Then)** 會完整且清晰地呈現深度技術文章的內容。

## 需求清單 _(必填)_

### 功能需求

- **FR-001**: 系統「必須」提供一個呈現工程師專業摘要、工作經驗總覽的首頁，且須包含常駐的履歷、LinkedIn 與 GitHub 連結。
- **FR-002**: 系統「必須」提供作品集區塊以展示 side project 列表，內容需包含標題、描述，並具備明確的行動呼籲 (Call to Action)，引導使用者前往 GitHub Repo 或 Live Demo。
- **FR-003**: 系統「必須」提供技術部落格區塊以條列已發布的文章。
- **FR-004**: 系統「必須」具備分類或標籤功能（例如：「前端開發」、「AI 應用」），讓使用者能輕鬆尋找特定主題的文章。
- **FR-005**: 系統「必須」具備完整的響應式設計 (Responsive Design)，確保在桌機與行動裝置上皆提供最佳的閱讀與互動體驗。

### 關鍵實體 (Key Entities) _(若涉及資料)_

- **Project (專案)**: 代表作品集內的成品。包含標題、簡短描述、技術堆疊標籤與外部網址（Demo、Repository）。
- **Article (文章)**: 代表部落格貼文。包含標題、發布日期、文章主體內容與主題標籤。

## 成功標準 _(必填)_

### 可量化成效

- **SC-001**: 訪客在首頁中只需不超過一次的點擊，即可存取履歷、LinkedIn 與 GitHub 連結。
- **SC-002**: 在一般寬頻網路環境下，作品集與部落格區塊的載入與內容顯示時間需小於 1.5 秒。
- **SC-003**: 本網站在標準的網頁效能與無障礙 (Accessibility) 測試工具中，無論是行動裝置或桌機版本，皆需獲得 90 分以上的評分。
- **SC-004**: 所有的專案外部連結與文章內部連結皆須能成功路由，不得發生 404 錯誤。
